# リポジトリ全体方針（Project Profile）

## 1. プロジェクトの目的
本リポジトリは、**「AIを活用した次世代のドキュメント作成・管理（Docs as Code）」**を実践・探求するためのショーケース兼実務ワークスペースです。
Marpを使用したプレゼンテーション資料作成を中心に、AI技術（エージェント、MCP、LLM等）を生産性向上に直結させる手法を蓄積し、言語化して発信・共有することを目的とします。

## 2. コア・コンセプト
本リポジトリの運用は、以下の3つのコア・コンセプトに基づきます。

### 1) Docs as Code（ドキュメントのコード化）
* すべてのスライドやドキュメントは、Markdown（プレーンテキスト）で記述し、Git/GitHubによるバージョン管理を行います。
* `marp.json` や `themes/` などの共通設定により、デザインの統一と作成の手間（DRY原則）を両立させます。
* 特定のソフト（PowerPoint等）に依存せず、どこでも誰でも編集可能であり、AIとの親和性が最も高いデータ形式を維持します。

### 2) Agentic Workflow（エージェント的ワークフロー）の体現
* AI（Antigravity等）を単なる「チャット相手（一問一答）」として扱うのではなく、「自律的なアシスタント」として活用します。
* プロジェクトごとに `1_Requirements` (要件)、`2_Knowledge` (知識)、`3_Output` (出力)、`.ai_settings` (AIルール) などの構造を用意し、AIがコンテキストを深く理解できる「環境」を作ります。

### 3) 継続的な資産化（使い捨てからの脱却）
* AIとの対話で得られた有用なプロンプトの型、リサーチ結果、思考のプロセス（Chain of Thought）は、すべて`.md`ファイルとしてリポジトリ内に永続化（ナレッジ化）します。
* こうして蓄積されたファイル群は、将来別のタスクをAIに依頼する際の「強力なコンテキスト集」として機能します。

## 3. 主要プロジェクト一覧（/projects）

このリポジトリ配下の `projects/` ディレクトリには、具体的なテーマごとのスライドやドキュメントが格納されています。

* **`ai_utilization/`**:
  * テーマ「一歩進んだAI活用術」
  * ビジネスパーソンや非エンジニア向けに、本リポジトリのコンセプトであるAgentic Workflowや自動化の哲学を解説した資料群です。
* **`ai_robot/`**:
  * テーマ「産業用ロボットのAI活用動向と次世代戦略」
  * Hailo-8等のエッジAIチップ実装を想定した、実用的な技術調査・戦略立案のスライドです。
* **`profile/`**:
  * 著者やプロジェクトオーナーのプロフィール、思想を発信するためのスライド資料です。

## 4. 運用ルール
1. **スライドの作成**: 各プロジェクトディレクトリ内で `slide.md` 等のMarkdownファイルを作成し、Marp CLI（またはVS Code拡張機能）でエクスポートしてください。
2. **AIとの協働**: 新しい資料を作成する際は、AIに対してこの `profile.md` や各フォルダの `.ai_settings` を参照させることで、的確なコンテキストを渡してください。
3. **コミットの粒度**: 意味のある単位でGitコミットを行い、変更履歴を追いやすくしてください。
